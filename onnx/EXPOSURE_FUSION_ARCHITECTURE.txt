================================================================================
PIPELINE POSITIONING
================================================================================

Exposure Fusion is a POST-PROCESS STAGE that operates AFTER YUV/RGB conversion.

Pipeline Order:
1. Demosaic
2. Black Level Correction
3. White Balance
4. CCM (Color Correction Matrix)
5. Gamma Correction
6. YUV/RGB Conversion
7. Exposure Fusion (HDR) ← POST-PROCESS STAGE
   ├─ ExposureFusionAlgo: Extract exposure info
   ├─ ExposureFusionLoop: Fuse multiple exposures
   └─ ExposureFusionApplier: Apply tone mapping
8. Output

Why Post-Process?
- Exposure fusion operates on final color-corrected frames
- Better color accuracy for exposure analysis
- Tone mapping should be applied after all color corrections
- Preserves color accuracy during fusion

================================================================================
CONTROL LOOP ARCHITECTURE
================================================================================

The control loop uses the ONNX Loop operator for iterative processing:

Loop Structure:
┌─────────────────────────────────────────────────────────┐
│                    Loop Operator                         │
│  ┌─────────────────────────────────────────────────┐  │
│  │              Loop Body (iterates N times)          │  │
│  │                                                   │  │
│  │  1. Calculate adaptive weights based on quality  │  │
│  │     weight = α * brightness + β * contrast + γ    │  │
│  │                                                   │  │
│  │  2. Update weights with fusion strength          │  │
│  │     w_new = w_old * (1-α) + quality * α           │  │
│  │                                                   │  │
│  │  3. Normalize weights                             │  │
│  │     w_norm = w_new / Σ(w_new)                     │  │
│  │                                                   │  │
│  │  4. Blend exposures using weights                 │  │
│  │     fused = Σ (w_norm[i] * frames[i])             │  │
│  │                                                   │  │
│  │  5. Update state for next iteration              │  │
│  │     state = {w_norm, fused}                        │  │
│  └─────────────────────────────────────────────────┘  │
│                                                         │
│  State Variables (persist across iterations):          │
│  - weights [m] : Weights for each exposure             │
│  - fused [n,3,h,w] : Current fused frame               │
└─────────────────────────────────────────────────────────┘

Weight Calculation:
  quality = (brightness + contrast + saturation) / 3
  w_new = w_old * (1-α) + quality * α
  
  Where:
  - α = fusion_strength (0-1)
  - brightness, contrast, saturation are normalized to [0, 1]

Ghosting Reduction (ExposureFusionLoopV1):
  - Detect motion between consecutive frames
  - Suppress ghosting artifacts
  - Use well-exposed mask to guide fusion

================================================================================
EXPOSURE FUSION ALGORITHM
================================================================================

Exposure Fusion Steps:

1. **Extract Exposure Information**
   - Calculate brightness per frame
   - Calculate contrast per frame
   - Calculate saturation per frame
   - Calculate quality score

2. **Calculate Weights**
   - Weight based on quality metrics
   - Normalize weights to sum to 1
   - Apply fusion strength

3. **Blend Exposures**
   - fused = Σ (weights[i] * frames[i])
   - Weighted average of all exposures

4. **Tone Mapping**
   - Normalize to [0, 1]
   - Apply gamma correction
   - Clip to valid range
   - Scale to output range

Quality Metrics:
  brightness = mean(frame)
  contrast = std(frame)
  saturation = max(frame) - min(frame)
  quality = (brightness + contrast + saturation) / 3

Well-Exposed Mask:
  well_exposed = |brightness - 0.5| < threshold
  
  Pixels with brightness close to 0.5 are considered well-exposed.

================================================================================
TONE MAPPING
================================================================================

Tone Mapping converts HDR to LDR for display:

Simple Tone Mapping (ExposureFusionApplierBase):
  normalized = fused / dynamic_range
  gamma_corrected = normalized^(1/γ)
  output = clip(gamma_corrected, 0, 1) * 255

Adaptive Tone Mapping (ExposureFusionApplierV1):
  exposure_comp = target_brightness / current_brightness
  exposed = fused * exposure_comp
  normalized = exposed / dynamic_range
  gamma_corrected = normalized^(1/γ)
  output = clip(gamma_corrected, 0, 1) * 255

Local Tone Mapping (ExposureFusionApplierV2):
  exposure_comp = target_brightness / current_brightness
  exposed = fused * exposure_comp
  luminance = mean(exposed)
  local_luminance = blur(luminance)
  luminance_adapted = (1-α) * luminance + α * local_luminance
  tone_mapped = exposed / luminance_adapted
  normalized = tone_mapped / dynamic_range
  gamma_corrected = normalized^(1/γ)
  output = clip(gamma_corrected, 0, 1) * 255

Gamma Correction:
  γ = 2.2 (standard gamma)
  output = input^(1/γ)

================================================================================
GHOSTING REDUCTION
================================================================================

Ghosting occurs when objects move between exposures:

Detection:
  diff = |frame[i] - frame[i+1]|
  ghost_mask = diff > threshold

Reduction:
  - Use well-exposed mask to guide fusion
  - Suppress weights for ghosted regions
  - Blend with motion compensation

Benefits:
  - Reduces ghosting artifacts
  - Improves fusion quality
  - Better for dynamic scenes
