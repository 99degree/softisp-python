LensGDCBase
-----------
Purpose: Base GDC implementation with external grid

Inputs:
- applier [n,3,h,w] : Image tensor from upstream
- gdc_grid [h,w,2]  : Geometric distortion correction grid (external)

Outputs:
- applier [n,3,h,w] : Geometrically corrected image tensor

Algorithm:
1. Receive external gdc_grid
2. Unsqueeze grid to [1,h,w,2] for GridSample
3. Apply GridSample for geometric transformation
4. Output corrected image

Use Case: When correction grid is provided by calibration or coordinator

GridSample Parameters:
- mode: bilinear
- padding_mode: zeros
- align_corners: 1

LensGDCV1
---------
Purpose: Radial distortion correction with polynomial model

Inputs:
- applier [n,3,h,w] : Image tensor from upstream
- gdc_params [4]    : [k1, k2, k3, vcm] distortion parameters

Outputs:
- applier [n,3,h,w] : Geometrically corrected image tensor
- gdc_grid [h,w,2]  : Generated correction grid

Algorithm:
1. Split parameters: k1, k2, k3, vcm
2. Create coordinate grids (normalized to [-1, 1])
3. Calculate radius squared: r^2 = x^2 + y^2
4. Calculate polynomial terms:
   - k1*r^2
   - k2*r^4
   - k3*r^6
5. Calculate scale: scale = 1 + k1*r^2 + k2*r^4 + k3*r^6 + vcm
6. Calculate corrected coordinates:
   - x' = x * scale
   - y' = y * scale
7. Stack into grid [h,w,2]
8. Apply GridSample for geometric transformation

Radial Distortion Model:
  r^2 = x^2 + y^2
  scale = 1 + k1*r^2 + k2*r^4 + k3*r^6 + vcm
  x' = x * scale
  y' = y * scale

Use Case: Correcting barrel/pincushion distortion in standard lenses

LensGDCV2
---------
Purpose: Fisheye distortion correction with advanced model

Inputs:
- applier [n,3,h,w] : Image tensor from upstream
- gdc_params [5]    : [k1, k2, k3, k4, vcm] distortion parameters

Outputs:
- applier [n,3,h,w] : Geometrically corrected image tensor
- gdc_grid [h,w,2]  : Generated correction grid

Algorithm:
1. Split parameters: k1, k2, k3, k4, vcm
2. Create coordinate grids (normalized to [-1, 1])
3. Calculate radius: r = sqrt(x^2 + y^2)
4. Calculate theta: theta = atan(r)
5. Calculate polynomial terms in theta:
   - theta^2, theta^4, theta^6, theta^8
   - k1*theta^2, k2*theta^4, k3*theta^6, k4*theta^8
6. Calculate polynomial correction:
   - poly = 1 + k1*theta^2 + k2*theta^4 + k3*theta^6 + k4*theta^8
7. Calculate r': r' = theta * poly
8. Calculate scale: scale = r' / r + vcm
9. Calculate corrected coordinates:
   - x' = x * scale
   - y' = y * scale
10. Stack into grid [h,w,2]
11. Apply GridSample for geometric transformation

Fisheye Distortion Model:
  r = sqrt(x^2 + y^2)
  theta = atan(r)
  r' = theta * (1 + k1*theta^2 + k2*theta^4 + k3*theta^6 + k4*theta^8)
  scale = r' / r + vcm
  x' = x * scale
  y' = y * scale

Use Case: Correcting fisheye distortion in wide-angle lenses

================================================================================
DISTORTION MODELS
================================================================================

Radial Distortion (V1)
----------------------
Corrects barrel and pincushion distortion in standard lenses.

Parameters:
- k1: First-order radial coefficient
- k2: Second-order radial coefficient
- k3: Third-order radial coefficient
- vcm: Vignetting correction multiplier

Typical Values:
- Barrel distortion: k1 > 0, k2 < 0, k3 > 0
- Pincushion distortion: k1 < 0, k2 > 0, k3 < 0

Fisheye Distortion (V2)
-----------------------
Corrects fisheye distortion in wide-angle lenses.

Parameters:
- k1: First-order fisheye coefficient
- k2: Second-order fisheye coefficient
- k3: Third-order fisheye coefficient
- k4: Fourth-order fisheye coefficient
- vcm: Vignetting correction multiplier

Typical Values:
- Fisheye: k1 > 0, k2 < 0, k3 > 0, k4 < 0

================================================================================
GRIDSAMPLE OPERATOR
================================================================================

The GridSample operator is used for geometric transformation:

Input:
- input: [n, c, h, w] - Input image tensor
- grid: [n, h_out, w_out, 2] - Sampling grid

Grid Format:
- grid[..., 0] = x coordinates (normalized to [-1, 1])
- grid[..., 1] = y coordinates (normalized to [-1, 1])

Parameters:
- mode: bilinear (interpolation mode)
- padding_mode: zeros (out-of-bounds handling)
- align_corners: 1 (coordinate system alignment)

Output:
- output: [n, c, h_out, w_out] - Transformed image

================================================================================
COMPARISON WITH LCS
================================================================================

LCS (Lens Shading Correction):
- Purpose: Correct vignetting and non-uniform illumination
- Method: Per-pixel gain multiplication
- Output: [n,3,h,w] corrected image
- Parameters: Gain map or radial parameters

GDC (Geometric Distortion Correction):
- Purpose: Correct geometric distortion (barrel/pincushion/fisheye)
- Method: GridSample-based geometric transformation
- Output: [n,3,h,w] corrected image
- Parameters: Distortion coefficients

Combined Usage:
1. Apply GDC first (geometric correction)
2. Apply LCS second (shading correction)
3. Order matters: GDC changes pixel positions, LCS adjusts pixel values

================================================================================
PERFORMANCE CONSIDERATIONS
================================================================================

GridSample Complexity:
- Time: O(n * c * h * w) - proportional to image size
- Space: O(h * w * 2) - grid storage

V1 Complexity:
- Time: O(h * w) - polynomial calculation per pixel
- Space: O(h * w * 2) - grid storage

V2 Complexity:
- Time: O(h * w) - polynomial + atan calculation per pixel
- Space: O(h * w * 2) - grid storage

Optimization Tips:
1. Pre-compute grids for fixed parameters
2. Use lower resolution for preview
3. Consider GPU acceleration for GridSample
