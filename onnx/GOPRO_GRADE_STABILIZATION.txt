================================================================================
STAGE 1: ALGO (Path Smoother with Horizon Leveling)
================================================================================

Purpose: Calculate smooth camera pose for cinematic stabilization

Input:
- current_frame [n,3,h,w] : Current video frame (YUV or RGB)
- prev_frame [n,3,h,w] : Previous video frame (YUV or RGB)
- gyro_data [3] : Raw gyroscope data [rx, ry, rz] (rad/s)
- gravity_vector [3] : Gravity vector [gx, gy, gz] (normalized)
- smoothing_factor [1] : Low-pass filter coefficient (0-1)
- horizon_leveling [1] : Enable horizon leveling (0=off, 1=on)

Output:
- homography [3,3] : Smoothed homography matrix (OpenCV format)
- rotation_matrix [3,3] : 3D rotation matrix (6-DOF)
- translation [3] : 3D translation vector [tx, ty, tz]
- dynamic_zoom [1] : Dynamic zoom factor (1.0 = no zoom)
- confidence [1] : Confidence score (0-1)

Class: DeshakeAlgoV2 (~40-50 ONNX nodes)

Hardware: NPU (Neural Processing Unit)

================================================================================
PATH SMOOTHING (Temporal Filtering)
================================================================================

The algo uses a low-pass filter to smooth the camera path:

    R_smooth[t] = α * R_smooth[t-1] + (1-α) * R_raw[t]

Where:
- R_smooth: Smoothed rotation matrix
- R_raw: Raw rotation from gyro
- α: Smoothing factor (higher = smoother but more lag)

This creates a cinematic path that removes high-frequency jitter
while preserving intentional camera movements.

================================================================================
HORIZON LEVELING (GoPro Hero Black Feature)
================================================================================

Calculates rotation needed to keep gravity vector at 0°:

    R_level = R_gravity * R_smooth

Where:
- R_gravity: Rotation to align gravity with vertical axis
- R_smooth: Smoothed camera rotation

Gravity Vector Processing:
    pitch = atan2(gx, gz)
    roll = atan2(-gy, sqrt(gx^2 + gz^2))

Rotation Matrix from Pitch and Roll:
    R_pitch = [[cos(p), 0, sin(p)],
               [0, 1, 0],
               [-sin(p), 0, cos(p)]]
    
    R_roll = [[1, 0, 0],
              [0, cos(r), -sin(r)],
              [0, sin(r), cos(r)]]
    
    R_level = R_roll * R_pitch

This is the hallmark of the GoPro Hero "Black" series - keeping
the horizon level even during extreme camera movements.

================================================================================
DYNAMIC ZOOM (Adaptive Crop)
================================================================================

Adjusts zoom based on shake severity:

    Z = 1.0 + β * |angular_velocity|

Where:
- Z: Zoom factor
- β: Zoom sensitivity
- angular_velocity: Magnitude of rotation

Benefits:
- Tighter crop = more stability
- Adaptive = less zoom when stable
- Cinematic = smooth transitions

================================================================================
STAGE 2: COORDINATOR (Rolling Shutter + 6-DOF)
================================================================================

Purpose: Generate row-dependent transformation for rolling shutter correction

Input:
- frame_homography [3,3] : Frame-based homography matrix
- rotation_matrix [3,3] : 3D rotation matrix (6-DOF)
- gdc_coeffs [4] : GDC coefficients [k1, k2, p1, p2]
- imu_dx [1] : IMU horizontal acceleration
- imu_dy [1] : IMU vertical acceleration
- imu_dr [1] : IMU angular velocity
- scan_time [1] : Rolling shutter scan time (seconds)
- gyro_delay [1] : Gyro delay (seconds)
- smoothing_factor [1] : Smoothing factor (0-1)
- iterations [1] : Number of smoothing iterations
- image_size [2] : [height, width] for grid generation
- mesh_size [2] : [mesh_h, mesh_w] for mesh-based warp (default: [32, 32])

Output:
- mesh_grid [mesh_h,mesh_w,2] : Mesh vertex grid for warp
- fused_grid [h,w,2] : Fused coordinate grid (optional, for debugging)
- homography [3,3] : Refined homography matrix
- velocity [3] : Estimated velocity [vx, vy, vr]

Class: DeshakeLoopV2 (~80-100 ONNX nodes)

Hardware: CPU / DSP

================================================================================
ROLLING SHUTTER CORRECTION (Jello Removal)
================================================================================

The sensor doesn't capture the whole image at once; it scans line-by-line.
If the camera moves during that scan, the image skews (jello effect).

To fix this, we generate a row-dependent transformation:

    R_row = Slerp(R_start, R_end, alpha)

Where:
- R_start: Camera pose at top of frame
- R_end: Camera pose at bottom of frame
- alpha: Row position (0 to 1)
- Slerp: Spherical Linear Interpolation

Row-Dependent Transformation:
    for v in range(height):
        alpha = v / (height - 1)
        R_row = interpolate_rotation(R_start, R_end, alpha)
        
        # Apply GDC + Rotation in 3D Space
        T = K * R_row * K_inv
        p_warped = T * p_distorted

This eliminates the "jello" effect common in fast-moving footage.

================================================================================
MESH-BASED WARP (32x32 Vertex Grid)
================================================================================

Instead of a per-pixel map, we output a 32x32 vertex grid.
The GPU uses these vertices to stretch the image like a texture
on a flexible 3D mesh.

Benefits:
- Much faster than per-pixel mapping
- Enables real-time 4K 60fps stabilization
- Smooth interpolation between vertices
- Hardware-accelerated on GPU

Mesh Grid Format:
    mesh_grid [32, 32, 2]
    
    Where:
    - mesh_grid[..., 0]: x coordinates (normalized to [-1, 1])
    - mesh_grid[..., 1]: y coordinates (normalized to [-1, 1])

================================================================================
TRANSFORMATION COMPOSITION (6-DOF)
================================================================================

The final coordinate P' for any pixel is the result of this chain:

    P' = K * R_row * K_inv * P_source

Where:
- K: Camera intrinsic matrix (focal length, principal point)
- R_row: Row-dependent rotation matrix
- P_source: Source pixel coordinates
- P_final: Final pixel coordinates for sampling

Step-by-Step:
1. Un-project: Turn 2D pixel into 3D ray (K_inv)
2. Un-distort: Remove the lens curve (GDC)
3. Rotate: Apply the anti-shake rotation (R_row)
4. Re-project: Turn the 3D ray back into 2D pixel (K)

================================================================================
STAGE 3: APPLIER (Mesh Warp with Edge Filling)
================================================================================

Purpose: Apply mesh-based warp for real-time 4K 60fps stabilization

Input:
- current_frame [n,3,h,w] : Current video frame (YUV or RGB)
- mesh_grid [mesh_h,mesh_w,2] : Mesh vertex grid from coordinator
- mesh_size [2] : [mesh_h, mesh_w] for mesh-based warp (default: [32, 32])
- padding_mode [1] : Padding mode (0=zeros, 1=border, 2=reflect)

Output:
- stabilized_frame [n,3,h,w] : Stabilized frame (YUV or RGB)
- valid_mask [n,1,h,w] : Valid region mask

Class: DeshakeApplierV2 (~20-30 ONNX nodes)

Hardware: GPU / Warp Hardware

================================================================================
EDGE FILLING (Hide Black Bars)
================================================================================

Uses "Reflect" or "Replicate" padding to hide the black bars
at the edge of the crop. This is the hallmark of GoPro's
Hypersmooth stabilization.

Padding Modes:
- 0 (zeros): Pad with black (default)
- 1 (border): Pad with edge pixels
- 2 (reflect): Mirror edge pixels (best for Hypersmooth)

GridSample Parameters:
- mode: 'bilinear' (bilinear interpolation)
- padding_mode: 'reflection' (reflect edge pixels)
- align_corners: 1 (align corners for better quality)
