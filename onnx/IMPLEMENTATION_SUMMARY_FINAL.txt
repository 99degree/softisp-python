================================================================================
ALGORITHMS CREATED
================================================================================

1. AutoExposureSimple (family: autoexposure_simple)
   - Computation: LOW
   - Use Case: Real-time, embedded, low-power
   - Features: Mean brightness, simple exposure calculation
   - ONNX Nodes: ~15

2. AutoExposureStats (family: autoexposure_stats)
   - Computation: MEDIUM
   - Use Case: General-purpose ISP, mobile, consumer cameras
   - Features: Per-channel RGB, BT.601 weighted luminance
   - ONNX Nodes: ~25

3. AutoExposureYUV (family: autoexposure_yuv)
   - Computation: MEDIUM
   - Use Case: YUV pipelines, video processing, streaming
   - Features: YUV luminance-only, chrominance preservation
   - ONNX Nodes: ~20

4. AutoExposureHistogram (family: autoexposure_histogram)
   - Computation: HIGH
   - Use Case: Professional cameras, high-quality imaging
   - Features: Histogram-based, percentile targeting
   - ONNX Nodes: ~20

5. AutoExposureMultiZone (family: autoexposure_multizone)
   - Computation: HIGH
   - Use Case: Advanced ISP, computational photography, HDR
   - Features: Multi-zone, center-weighted exposure
   - ONNX Nodes: ~20

================================================================================
KEY FEATURES
================================================================================

✅ Independent Classes
   - Each algorithm only depends on MicroblockBase
   - No dependencies on other auto exposure classes
   - No inheritance chain between algorithms

✅ Unique Family Names
   - autoexposure_simple
   - autoexposure_stats
   - autoexposure_yuv
   - autoexposure_histogram
   - autoexposure_multizone

✅ No Config Classes
   - All parameters are class attributes
   - No separate configuration classes
   - Simple and direct parameter access

✅ No __init__ Required
   - Classes work without instantiation
   - All parameters are class-level attributes
   - No initialization dependencies

✅ Shared Applier Logic
   - All algorithms use the same applier implementation
   - Multiply by gain, clip to [0, 1]
   - Consistent output format

✅ Required Methods
   - build_algo() - Calculate statistics
   - build_applier() - Apply exposure compensation
   - build_coordinator() - Coordinator interface
   - build_test_algo() - Test algorithm interface

================================================================================
EXTENSIBILITY
================================================================================

To add a new algorithm:

1. Create a new class in autoexposure_v2.py:

class AutoExposureCustom(MicroblockBase):
    name = "autoexposure_custom"
    family = "autoexposure_custom"
    version = "v2"
    
    target_brightness = 0.18
    min_ev = -2.0
    max_ev = 2.0
    
    def build_algo(self, stage, prev_stages=None):
        # Your custom algorithm here
        pass
    
    def build_applier(self, stage, prev_stages=None):
        # Your applier here
        pass
    
    def build_coordinator(self, stage, prev_stages=None):
        return super().build_coordinator(stage, prev_stages)
    
    def build_test_algo(self, stage, prev_stages=None):
        return self.build_algo(stage, prev_stages)

2. Add to __init__.py exports:

from .autoexposure_v2 import AutoExposureCustom

__all__ = [
    'AutoExposureSimple',
    'AutoExposureStats',
    'AutoExposureYUV',
    'AutoExposureHistogram',
    'AutoExposureMultiZone',
    'AutoExposureCustom'
]
