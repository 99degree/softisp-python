================================================================================
STAGE 1: ALGO DOMAIN
================================================================================

Purpose: Extract stats and output homography 3x3 matrix.

Classes:
- DeshakeAlgoBase: Simple block matching, ~15-20 ONNX nodes
- DeshakeAlgoV1: Grid-based motion estimation, ~30-40 ONNX nodes
- DeshakeAlgoV2: Feature-based with RANSAC, ~50-60 ONNX nodes

Input:
- current_frame [n,3,h,w] : Current video frame (YUV or RGB)
- prev_frame [n,3,h,w] : Previous video frame (YUV or RGB)

Output:
- homography [3,3] : 3x3 homography matrix (OpenCV format)
- confidence [1] : Confidence score (0-1)

Behavior:
- build_algo: Extracts homography matrix from frames
- build_coordinator: None (not used)
- build_applier: None (not used)

Homography Matrix Format (OpenCV):
    H = [h00 h01 h02]
        [h10 h11 h12]
        [h20 h21 h22]
    
    Where:
    - h00, h01, h10, h11: Rotation and scaling
    - h02, h12: Translation (tx, ty)
    - h20, h21: Perspective (shear)
    - h22: Scale (usually 1)

================================================================================
STAGE 2: COORDINATOR DOMAIN (Mesh Grid Generation)
================================================================================

Purpose: Generate mesh grid from homography matrix.

Classes:
- DeshakeMeshGridBase: Base mesh grid generator, ~40-50 ONNX nodes
- DeshakeMeshGridSimple: Simple GDC + homography fusion, ~40-50 ONNX nodes
- DeshakeMeshGridFull: Full IMU fusion with temporal smoothing, ~60-80 ONNX nodes

Input:
- homography [3,3] : 3x3 homography matrix from algo
- camera_matrix [3,3] : 3x3 camera intrinsic matrix K
- gdc_coeffs [4] : GDC coefficients [k1, k2, p1, p2]
- mesh_size [2] : [mesh_h, mesh_w] for mesh-based warp (default: [16, 16])
- image_size [2] : [height, width] for grid generation

Full Implementation Additional Inputs:
- imu_rotation [3,3] : 3x3 IMU rotation matrix (from CPU coordinator)
- smoothing_alpha [1] : Temporal smoothing factor (0.0-1.0)
- horizon_leveling [1] : Enable horizon leveling (0=off, 1=on)
- dynamic_zoom [1] : Enable dynamic zoom (0=off, 1=on)
- rolling_shutter [1] : Enable rolling shutter correction (0=off, 1=on)

Output:
- mesh_grid [mesh_h,mesh_w,2] : Mesh vertex grid for warp
- valid_mask [mesh_h,mesh_w] : Valid region mask

Behavior:
- build_algo: None (not used)
- build_coordinator: Generate mesh grid from homography
- build_applier: None (not used)

Mesh Grid Generation Steps:
1. Create normalized identity grid
2. Apply GDC distortion to grid points
3. Apply homography transformation
4. Output mesh grid for GPU warp

Transformation Composition:
    P_final = K * H * K_inv * P_source
    
    Where:
    - K: Camera intrinsic matrix
    - H: Homography matrix from algo
    - P_source: Source pixel coordinates
    - P_final: Final pixel coordinates for sampling

Full Implementation Features:
- Sensor Fusion: R_fused = α * R_homography + (1-α) * R_IMU
- Temporal Smoothing: R_smooth[t] = α * R_smooth[t-1] + (1-α) * R_fused[t]
- Horizon Leveling: R_level = R_gravity * R_smooth
- Dynamic Zoom: Z = 1.0 + β * |angular_velocity|
- Rolling Shutter Correction: R_row = Slerp(R_start, R_end, alpha)

================================================================================
STAGE 3: APPLIER DOMAIN (Grid Mapping)
================================================================================

Purpose: Apply grid mapping to frame using mesh grid coefficients.

Classes:
- DeshakeApplier: Simple grid mapping, ~15-20 ONNX nodes
- DeshakeApplierV2: Advanced grid mapping with edge filling, ~20-30 ONNX nodes

Input:
- current_frame [n,3,h,w] : Current video frame (YUV or RGB)
- mesh_grid [mesh_h,mesh_w,2] : Mesh vertex grid from coordinator
- image_size [2] : [height, width] for upsampling
- padding_mode [1] : Padding mode (0=zeros, 1=border, 2=reflect)

Output:
- stabilized_frame [n,3,h,w] : Stabilized frame (YUV or RGB)
- valid_mask [n,1,h,w] : Valid region mask

Behavior:
- build_algo: None (not used)
- build_coordinator: None (not used)
- build_applier: Apply grid mapping using mesh grid coefficients

Grid Mapping:
    Uses mesh grid coefficients to warp the image. The mesh grid
    contains the sampling coordinates for each pixel.
    
    Benefits:
    - Hardware-accelerated on GPU
    - Single-pass operation
    - Professional-grade quality

Edge Filling:
    Uses "Reflect" or "Replicate" padding to hide the black bars
    at the edge of the crop.

PERFORMANCE CHARACTERISTICS
================================================================================

Stage 1: Algo Domain
- DeshakeAlgoBase: ~15-20 ONNX nodes
- DeshakeAlgoV1: ~30-40 ONNX nodes
- DeshakeAlgoV2: ~50-60 ONNX nodes

Stage 2: Coordinator Domain
- DeshakeMeshGridBase: ~40-50 ONNX nodes
- DeshakeMeshGridSimple: ~40-50 ONNX nodes
- DeshakeMeshGridFull: ~60-80 ONNX nodes

Stage 3: Applier Domain
- DeshakeApplier: ~15-20 ONNX nodes
- DeshakeApplierV2: ~20-30 ONNX nodes

Total Complexity:
- Simple Pipeline (Base + Simple + Applier): ~70-90 ONNX nodes
- Full Pipeline (V2 + Full + V2): ~130-170 ONNX nodes

Real-Time Performance:
- 1080p 60fps: ~3-5ms per frame (Simple), ~6-10ms per frame (Full)
- 4K 60fps: ~8-12ms per frame (Simple), ~15-25ms per frame (Full)
- 4K 120fps: ~15-20ms per frame (Simple), ~30-40ms per frame (Full)

================================================================================
KEY BENEFITS
================================================================================

✅ **Clear Stage Separation**
   - Stage 1: Algo Domain → Output homography [3,3]
   - Stage 2: Coordinator Domain → Output mesh_grid [mesh_h,mesh_w,2]
   - Stage 3: Applier Domain → Apply grid mapping to frame

✅ **Industrial-Level API**
   - Each stage has clear responsibility
   - Easy to swap implementations
   - Modular and flexible

✅ **Mesh Grid as Coefficients**
   - Coordinator outputs mesh grid
   - Applier takes mesh grid as coefficients
   - Clean separation of concerns

✅ **Two Complementary Implementations**
   - Simple: GDC + homography fusion
   - Full: IMU fusion with temporal smoothing

✅ **Hardware Optimization**
   - Algo on NPU (feature extraction)
   - Coordinator on CPU/DSP (mesh generation)
   - Applier on GPU (grid mapping)

✅ **Flexible Architecture**
   - Works with or without IMU data
   - Adapts to different mesh sizes
   - Supports multiple padding modes
