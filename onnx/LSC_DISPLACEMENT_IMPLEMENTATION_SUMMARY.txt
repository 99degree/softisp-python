VCM POSITION CONTEXT
================================================================================

What is VCM (Voice Coil Motor)?
--------------------------------
VCM is the actuator that moves the lens for autofocus. The VCM position
determines the lens focal length and affects lens characteristics including:
- Vignetting (lens shading)
- Geometric distortion
- Depth of field

VCM Position Range:
- VCM position changes from min to max during autofocus operation
- Different VCM positions have different lens characteristics
- LCS coefficients may vary with VCM position

VCM Position in Polynomial:
- VCM position is used as a base offset in the polynomial
- Accounts for lens position-dependent characteristics
- Different VCM positions may have different LCS coefficients (k1, k2, k3)

LensLCSDisplacementBase
-----------------------
Purpose: Base LCS implementation with external displacement map

Inputs:
- applier [n,3,h,w] : Image tensor from upstream
- displacement_map [h,w,2] : Per-pixel displacement vectors (external)
- lsc_gain [h,w] : Per-pixel gain factors (external)

Outputs:
- applier [n,3,h,w] : Corrected image tensor

Algorithm:
1. Receive external displacement_map and lsc_gain
2. Split displacement map into dx and dy
3. Create coordinate grids (normalized to [-1, 1])
4. Apply displacement: x_sample = x + dx, y_sample = y + dy
5. Stack into grid [h,w,2]
6. Apply GridSample for geometric transformation
7. Apply gain correction
8. Output corrected image

Use Case: When displacement map and gain are provided by calibration

GridSample Parameters:
- mode: bilinear
- padding_mode: zeros
- align_corners: 1

LensLCSDisplacementV1
---------------------
Purpose: Displacement map generation from standard LCS coefficients and VCM position

Inputs:
- applier [n,3,h,w] : Image tensor from upstream
- lsc_coeffs [2] : [k1, k2] standard LCS coefficients
- vcm_pos [1] : VCM position (lens focal length parameter)

Outputs:
- applier [n,3,h,w] : Corrected image tensor
- displacement_map [h,w,2] : Generated displacement vectors
- lsc_gain [h,w] : Generated gain factors

Algorithm:
1. Split LCS coefficients: k1, k2
2. Create coordinate grids (normalized to [-1, 1])
3. Calculate radius squared: r^2 = x^2 + y^2
4. Calculate polynomial terms (standard LCS model):
   - k1*r^2
   - k2*r^4
5. Calculate scale: scale = k1*r^2 + k2*r^4 + vcm_pos
6. Calculate displacement vectors:
   - dx = x * scale
   - dy = y * scale
7. Stack into displacement map [h,w,2]
8. LSC gain = scale^2
9. Apply GridSample for geometric transformation
10. Apply gain correction

Standard LCS Polynomial Model with VCM Position:
  r^2 = x^2 + y^2
  poly_term = k1*r^2 + k2*r^4
  scale = poly_term + vcm_pos
  dx = x * scale
  dy = y * scale
  gain = scale^2

Use Case: Standard LCS displacement correction with 2 coefficients

VCM Position Context:
- VCM position determines lens focal length
- VCM position changes from min to max during autofocus
- Different VCM positions may have different k1, k2 coefficients
- Polynomial accounts for lens position-dependent characteristics

STANDARD LCS PARAMETERS
================================================================================

What are Standard LCS Parameters?
----------------------------------
Standard LCS parameters are the coefficients used in traditional lens shading
correction to model vignetting and radial gain patterns.

Parameters:
- k1: First-order radial coefficient
- k2: Second-order radial coefficient
- k3: Third-order radial coefficient (optional)
- vcm_pos: VCM position (lens focal length parameter)

Polynomial Model:
  scale = k1*r^2 + k2*r^4 + k3*r^6 + vcm_pos

Where:
- r^2 = x^2 + y^2 (normalized coordinates)
- scale: Per-pixel scale factor
- vcm_pos: VCM position (lens focal length)

Typical Values:
- k1: 0.01 to 0.1 (first-order vignetting)
- k2: -0.0001 to -0.001 (second-order correction)
- k3: 0.000001 to 0.00001 (third-order correction)
- vcm_pos: 0.0 to 10.0 (VCM position range)

VCM Position Range:
- VCM position changes from min to max during autofocus
- Different VCM positions have different lens characteristics
- LCS coefficients may vary with VCM position

================================================================================
DISPLACEMENT MAP CONCEPTS
================================================================================

What is a Displacement Map?
----------------------------
A displacement map is a 2D array where each pixel contains a 2D vector (dx, dy)
that specifies how far to shift that pixel in the source image to get the
corrected pixel in the output image.

Displacement Map Format:
- Shape: [h, w, 2]
- Channel 0: dx (horizontal displacement)
- Channel 1: dy (vertical displacement)
- Values: Normalized to image coordinates

Why Use Displacement Maps?
--------------------------
1. Per-pixel precision: Each pixel can have its own correction
2. Complex distortions: Can model non-radial distortions
3. Calibration-based: Can be derived from actual lens measurements
4. Flexibility: Can combine multiple correction types

Displacement vs. Traditional LCS:
---------------------------------
Traditional LCS:
- Only applies gain correction
- No geometric correction
- Simple multiplication

Displacement-based LCS:
- Applies both geometric and gain correction
- Uses GridSample for pixel resampling
- More accurate for complex lens distortions

================================================================================
GRIDSAMPLE OPERATOR
================================================================================

The GridSample operator is used for geometric transformation:

Input:
- input: [n, c, h, w] - Input image tensor
- grid: [n, h_out, w_out, 2] - Sampling grid

Grid Format:
- grid[..., 0] = x coordinates (normalized to [-1, 1])
- grid[..., 1] = y coordinates (normalized to [-1, 1])

Parameters:
- mode: bilinear (interpolation mode)
- padding_mode: zeros (out-of-bounds handling)
- align_corners: 1 (coordinate system alignment)

Output:
- output: [n, c, h_out, w_out] - Transformed image

VCM POSITION AND LENS CHARACTERISTICS
================================================================================

How VCM Position Affects Lens Characteristics:
----------------------------------------------
1. Vignetting: Different VCM positions have different vignetting patterns
2. Geometric Distortion: Lens distortion varies with focal length
3. Depth of Field: Changes with lens position
4. Field of View: Changes with focal length

VCM Position Range:
- VCM position changes from min to max during autofocus
- Min position: Closest focus distance
- Max position: Farthest focus distance
- Typical range: 0 to 10 (normalized)

LCS Coefficients vs VCM Position:
---------------------------------
Different VCM positions may have different LCS coefficients:
- k1, k2, k3 may vary with VCM position
- Calibration data may include coefficient tables
- Interpolation may be used between VCM positions

Practical Implementation:
1. Calibrate LCS coefficients at multiple VCM positions
2. Store coefficient lookup table
3. Interpolate coefficients based on current VCM position
4. Apply polynomial with interpolated coefficients

PERFORMANCE CONSIDERATIONS
================================================================================

GridSample Complexity:
- Time: O(n * c * h * w) - proportional to image size
- Space: O(h * w * 2) - displacement map storage

V1 Complexity:
- Time: O(h * w) - polynomial calculation per pixel
- Space: O(h * w * 2) - displacement map storage

V2 Complexity:
- Time: O(h * w) - extended polynomial per pixel
- Space: O(h * w * 2) - displacement map storage

VCM Position Update Frequency:
- VCM position changes during autofocus
- May need to update displacement map frequently
- Consider caching for static VCM positions
- Consider GPU acceleration for real-time updates

Optimization Tips:
1. Pre-compute displacement maps for fixed VCM positions
2. Use lower resolution for preview
3. Consider GPU acceleration for GridSample
4. Cache displacement maps when possible
5. Interpolate between pre-computed displacement maps
