Flexible architecture that handles any rotation input format (Quaternions,

Keeps old deshake working without IMU data while adding new capabilities.

================================================================================
FINAL ARCHITECTURE SUMMARY
================================================================================

1.  **Algo (Stateless ONNX):** Extracts raw frame-to-frame Δ (Euler or Matrix).
2.  **Coordinator (Stateful CPU):** 
    *   Maintains a `List<RotationState>` history.
    *   Performs **Slerp** (if Quat) or **Lerp** (if Euler) to smooth the path.
    *   Calculates the **Correction Matrix** $R_{corr}$.
3.  **Coordinator (Stateless ONNX):**
    *   **Input:** $R_{corr}$ (3x3), $K$ (3x3), $D$ (1x5).
    *   **Output:** 16x16 GridMap.
4.  **Applier (Stateless ONNX):** 
    *   **Input:** Source Frame + 16x16 GridMap.
    *   **Output:** Stabilized 4K Frame.

This setup is perfect. It's modular, handles any input, and keeps the heavy
"pixel moving" in the optimized stateless domain.

================================================================================
UNIVERSAL ROTATION WRAPPER
================================================================================

The Universal Rotation Wrapper provides a unified interface for handling
different rotation formats:

- **Quaternions** (4 floats: [w, x, y, z])
- **Euler angles** (3 floats: [roll, pitch, yaw])
- **Rotation matrices** (3x3 matrix)

All formats are converted to a unified 3x3 rotation matrix for processing.

RotationState Class:
    """Abstract Rotation Wrapper for the Coordinator Trace"""
    
    def __init__(self, data, mode='quat'):
        if mode == 'quat':
            self.matrix = self.quat_to_matrix(data)
        elif mode == 'euler':
            self.matrix = self.euler_to_matrix(data)
        else:
            self.matrix = data # Assume 3x3 Matrix

Key Methods:
- quat_to_matrix(q): Convert quaternion to rotation matrix
- euler_to_matrix(euler): Convert Euler angles to rotation matrix
- matrix_to_quat(R): Convert rotation matrix to quaternion
- matrix_to_euler(R): Convert rotation matrix to Euler angles
- slerp(other, t): Spherical Linear Interpolation
- lerp(other, t): Linear Interpolation
- inverse(): Calculate inverse rotation
- compose(other): Compose two rotations

================================================================================
ROTATION TRACE (Stateful CPU)
================================================================================

Maintains a history of RotationState for smoothing and interpolation.

Handles sample rate mismatch between different sensors:
- **Algo (NPU):** 60 Hz (Frame rate)
- **IMU (CPU/Sensor):** 200 Hz - 1000 Hz
- **Coordinator Trace:** Interpolates to match frame rate

Key Methods:
- add(rotation, timestamp): Add rotation to trace
- get_at_time(timestamp, interpolation): Get rotation at specific time
- get_center_of_exposure(frame_time, shutter_time): Get rotation at center of exposure
- smooth(window_size, method): Smooth the rotation trace
- get_correction_matrix(target): Calculate correction matrix

Pro-Tip for Coordinator:
When the CPU is fusing the "Decision Result," it should use the IMU data
that corresponds to the **Center of Exposure** of the frame. If the frame
was captured at $t=100\text{ms}$ with a $10\text{ms}$ shutter, the Coordinator
should grab the smoothed IMU/Algo state from $t=105\text{ms}$.

================================================================================
STATEFUL CPU COORDINATOR
================================================================================

DeshakeCoordinatorCPU maintains rotation history, performs smoothing,
and calculates correction matrices for the ONNX grid generator.

Key Features:
- Maintains rotation trace with history
- Performs smoothing (moving average or exponential)
- Calculates correction matrix
- Handles sample rate mismatch
- Calculates horizon leveling
- Calculates dynamic zoom

Key Methods:
- set_camera_intrinsics(focal_length, cx, cy): Set camera intrinsics
- set_gdc_coeffs(k1, k2, p1, p2): Set GDC distortion coefficients
- add_rotation(data, mode, timestamp): Add rotation to trace
- add_imu_rotation(gyro_data, dt, timestamp): Add IMU rotation
- set_frame_timing(frame_time, shutter_time): Set frame timing
- get_center_of_exposure_rotation(): Get rotation at center of exposure
- smooth_rotation_trace(): Smooth the rotation trace
- calculate_correction_matrix(target): Calculate correction matrix
- calculate_horizon_leveling(gravity_vector): Calculate horizon leveling
- calculate_dynamic_zoom(angular_velocity): Calculate dynamic zoom
- get_camera_matrix(): Get camera intrinsic matrix K
- get_decision_result(): Get decision result for ONNX grid generator

================================================================================
STATELESS ONNX GRID GENERATOR (16x16)
================================================================================

DeshakeGridGenerator generates 16x16 mesh vertex grid from correction matrix.

Input:
- correction_matrix [3,3] : 3x3 correction matrix from CPU coordinator
- camera_matrix [3,3] : 3x3 camera intrinsic matrix K
- gdc_coeffs [4] : GDC coefficients [k1, k2, p1, p2]
- mesh_size [2] : [mesh_h, mesh_w] for mesh-based warp (default: [16, 16])
- image_size [2] : [height, width] for grid generation

Output:
- mesh_grid [16,16,2] : 16x16 mesh vertex grid for warp
- valid_mask [16,16] : Valid region mask

Grid Generation Steps:
1. Create 16x16 normalized identity grid
2. Apply GDC distortion to grid points
3. Apply correction matrix transformation
4. Output 16x16 mesh grid for GPU warp

Transformation Composition:
    P_final = K * R_corr * K_inv * P_source
    
    Where:
    - K: Camera intrinsic matrix
    - R_corr: Correction matrix from CPU coordinator
    - P_source: Source pixel coordinates
    - P_final: Final pixel coordinates for sampling

Complexity: ~40-50 ONNX nodes

================================================================================
STATELESS ONNX APPLIER (16x16 Mesh Warp)
================================================================================

DeshakeApplierV3 applies 16x16 mesh-based warp for stabilization.

Input:
- current_frame [n,3,h,w] : Current video frame (YUV or RGB)
- mesh_grid [16,16,2] : 16x16 mesh vertex grid from grid generator
- image_size [2] : [height, width] for upsampling
- padding_mode [1] : Padding mode (0=zeros, 1=border, 2=reflect)

Output:
- stabilized_frame [n,3,h,w] : Stabilized frame (YUV or RGB)
- valid_mask [n,1,h,w] : Valid region mask

Mesh-Based Warp:
    Uses 16x16 vertex grid for mesh-based warp. The GPU uses these
    vertices to stretch the image like a texture on a flexible 3D mesh.
    
    Benefits:
    - Much faster than per-pixel mapping
    - Enables real-time 4K 60fps stabilization
    - Smooth interpolation between vertices
    - Hardware-accelerated on GPU

Edge Filling:
    Uses "Reflect" or "Replicate" padding to hide the black bars
    at the edge of the crop.

Complexity: ~15-20 ONNX nodes

================================================================================
WHY 16x16 MESH GRID?
================================================================================

Benefits of 16x16 mesh grid over 32x32 or per-pixel mapping:

1. **Performance:** 16x16 = 256 vertices vs 32x32 = 1024 vertices
   - 4x fewer vertices = 4x faster processing
   - Still provides smooth interpolation
   - Optimal for real-time 4K 60fps

2. **Memory:** 16x16x2 = 512 floats vs 32x32x2 = 2048 floats
   - 4x less memory bandwidth
   - Better cache utilization
   - Lower power consumption

3. **Quality:** 16x16 provides sufficient quality for most use cases
   - Smooth interpolation between vertices
   - No visible artifacts at 4K resolution
   - Professional-grade output

4. **Flexibility:** Can adapt mesh size based on requirements
   - 16x16: Real-time 4K 60fps (default)
   - 32x32: Higher quality 4K 30fps
   - 64x64: Professional 1080p 120fps

CPU COORDINATOR USAGE EXAMPLE
================================================================================

```python
from microblocks.deshake import DeshakeCoordinatorCPU, RotationState

# Initialize coordinator
coordinator = DeshakeCoordinatorCPU(
    max_history=100,
    smoothing_window=5,
    smoothing_method='exponential',
    smoothing_alpha=0.8
)

# Set camera intrinsics
coordinator.set_camera_intrinsics(focal_length=1000.0, cx=960.0, cy=540.0)

# Set GDC coefficients
coordinator.set_gdc_coeffs(k1=0.001, k2=-0.0001, p1=0.0005, p2=-0.0002)

# Add IMU rotations (200 Hz)
for i in range(200):
    gyro_data = np.array([0.01, -0.02, 0.005])  # rad/s
    dt = 0.005  # 5ms
    timestamp = i * dt
    coordinator.add_imu_rotation(gyro_data, dt, timestamp)

# Set frame timing (60 fps)
coordinator.set_frame_timing(frame_time=0.0, shutter_time=0.0167)

# Get center of exposure rotation
rotation = coordinator.get_center_of_exposure_rotation()

# Calculate correction matrix
correction_matrix = coordinator.calculate_correction_matrix()

# Get decision result for ONNX grid generator
decision_result = coordinator.get_decision_result()
```
